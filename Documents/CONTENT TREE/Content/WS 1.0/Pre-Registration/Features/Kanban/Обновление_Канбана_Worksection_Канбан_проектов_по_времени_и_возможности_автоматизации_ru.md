## Канбан проектов

Теперь вам доступно **отображение проектов в формате Канбана:**

​​​​​​​​​​​​​​​​​​​​​

![](/images/upload/Kanban_2_eng_.@x1_1.png)  

Этот вид отображения четко показывает, **на каком этапе находится конкретный
проект компании.** Особенно удобным этот функционал будет для типов бизнеса,
которые создают проект в системе под конкретного клиента или продукт:
_маркетинговые и диджитал агентства, продуктовые ИТ компании, строительства и
т.д._

### ​​​​Суть обновления

​Ранее была доступна фильтрация проектов по их статусам. Но это не давало
возможности отображать проекты, статусы которых не попадали в выборку. Поэтому
мы решили **найти и предложить вам новое решение.**

​​​​​​​​​​​​​​​​​​​

**![](/images/upload/en_view_switch_kanbanprojects.@x1_1.gif)​​​​​​​​​​​​​​​​​​​​**

​​​​​​​​​​​​​​​​​​

​​​​​​​​​​​​​​​​​​

С помощью отображения проектов в виде Канбана мы расширили ваши возможности.
Теперь информация об этапах каждого из проектов**собрана в одном месте и
сгруппирована по статусам**. Проекты же не имеющие статуса будут отображаться
в бэклоге, из которого вы удобно можете их переместить, когда над проектом
начнется работа.

  

![](/images/upload/en_project_kanban_overview.@x1_1.gif)​​​​​​​​​​​

###### **​ Важно!** Функция статусов и меток проектов доступна на тарифах
[премиум и выше. ](https://worksection.com/price.html)​

### Канбан проектов для повышения эффективности

Благодаря этому вы четко можете разделять нагрузку между проектами. Ведь

отвечает на актуальные организационные вопросы: сколько проектов на этапе
ожидания, сколько из них началось, а какие сейчас уже подходят к концу.

## Канбан по срокам

Мы позаботились и об **улучшении организации работы** каждого пользователя.
Теперь каждый работник будет иметь **основную рабочую страницу** с
возможностью работать по своим задачам и одновременно видеть, что
запланировано на будущее. Это поможет оперативно планировать нагрузку и
держать текущие задачи под контролем.

  

![](https://worksection.com/images/upload/Kanban_5_eng_.@x1.png)

  

До обновления пользователь мог контролировать задачи применяя параметры
фильтра по сроку: «просроченные», «сегодня», «на завтра» и другие. Сейчас
пользователь может отобразить свою нагрузку **в одном рабочем поле.**

### Суть обновления

**Группирование происходит на основе сроков текущих задач пользователя.**
Таким образом, работники четко видят сколько у них задач без срока, сколько
просроченных, сколько из них запланировано на сегодня, а какие должны быть
выполнены в ближайшее время.

  

![](/images/upload/Kanban_6_eng_.@x1_1.png)​​​​​​​​

  

Задачи также можно с**легкостью перемещать по Канбану** изменяя сроки при
условии наличия соответствующих прав. При перемещении система позволяет
выбрать конкретный день и подсвечивает выходные дни – все для удобного
планирования задач на следующие недели.

  

![](/images/upload/en_bydays_overview.@x1_1.gif)​​​​​​​

## Новые автоматизации для Канбана

К упомянутым выше обновлениям мы также добавили расширение возможностей
автоматизации, о которых часто просили наши пользователи:

### ![](https://cdn.jsdelivr.net/npm/emoji-datasource-
apple@14.0.0/img/apple/64/0031-fe0f-20e3.png) Установка срока на конкретный
статус

Теперь вы можете**задавать сроки** для выполнения задачи конкретного статуса.

  

![](/images/upload/en_kanban_menu_statusdeadline.@x1_1.gif)  

Также у вас есть возможность **задать определенные условия** при установлении
сроков на статус:

  

![](/images/upload/Kanban_9_eng_.@x1_1.png)​​​​​​

  1. **Установить новый срок.** Это может быть полезным, если статусами вы обозначаете определенные рабочие процессы, которые имеют нормированное количество дней для их выполнения.
  2. **Сдвинуть вперед –** сместить дедлайн на N количество дней при попадании на соответствующий статус. Например, при перемещении задачи не этап со статусом «На утверждение» дедлайн автоматически будет смещаться на заданное вами количество дней. 
  3. **Очистить,** то есть оставить задачу без дедлайна. Полезно для отметки задач, которые не требуют контроля сроков на дальнейших этапах.
  4. **Не менять –** оставить установленный ранее срок выполнения без изменений.

​​​​​​

Эти удобные варианты настроек сроков на конкретный статус помогут
автоматизировать планирование рабочего процесса и оптимизировать ваш
менеджмент.

### ![](https://cdn.jsdelivr.net/npm/emoji-datasource-
apple@14.0.0/img/apple/64/0032-fe0f-20e3.png) Назначение ответственного за
статус

Выберите кто из команды будет отвечать **за конкретный статус выполнения
задания:**

  

![](/images/upload/Kanban_11_eng_.@x1_1.png)  

#### [Первый ответственный]

**Первый ответственный -** ответственный на момент создания задачи.
Пригодится, если задача после прохождения нескольких этапов с разными
ответственными должна вернуться к первому исполнителю.

  

![](/images/upload/Kanban_12_eng_.@x1_2.png)  

**Например,** руководитель создает в системе задачу на СММ-менеджера. Он в
свою очередь для успешного выполнения должен сформировать техническое задание
на копирайтера и дизайнера, где каждый специалист отвечает за свой этап. После
завершения каждым своей части работы задача получает статус «Готово», за
которым закреплен первый исполнитель. Далее СММ-менеджер с уже полученными
материалами может публиковать сообщение в соцсетях.

#### [Предварительный ответственный]

**Предыдущий ответственный -** ответственный на момент предыдущей
автоматической смены. Удобно использовать для взаимозависимых рабочих
процессов.

  

![](/images/upload/Kanban_13_eng_.@x1_1.png)  

**Например,** программист выполняет задачу с трекингом времени и отдает
результат на тестирование. После теста задача возвращается автоматически на
программиста при установлении соответствующего статуса. Таким образом, в
отчетах будет сохраняться реальный исполнитель, что важно для контроля
использованных ресурсов.

#### [Автор задачи]

**Автор задачи -** пользователь, создавший данную задачу. Отлично подходит для
этапа утверждений и согласований рабочих процессов.

### ![](https://cdn.jsdelivr.net/npm/emoji-datasource-
apple@14.0.0/img/apple/64/1f454.png) Опыт клиентов

Наш клиент поделился удобством обновления на реальном примере
усовершенствования рабочего процесса для согласования отпусков.

  

**Ранее** он требовал прямого привлечения HR-менеджера для контроля
прохождения всех этапов, что снижало общую эффективность. С новыми
возможностями автоматизации администрирования процесса принятия отпуска
**делегировали непосредственно Worksection.**

  

###### _Работник ставит задачу на HR-менеджера с запросом на отпуск. Далее HR-
менеджер проверяет количество доступных дней и передает задачу на согласование
руководителю. По результатам рассмотрения запроса на отпуск задача получит
соответствующий статус «Согласовано руководителем» или «Не согласовано
руководителем» и возвращается к ознакомлению автору задачи._

![](/images/upload/Kanban_14_eng_.@x1_1.png)  

**Таким образом,** Worksection помог оптимизировать административный процесс и
разгрузить ресурсы команды для выполнения более приоритетных задач.
